## Strategy Overview

The task is to build a **full detection pipeline from scratch**: raw aerial video → auto-labeling → training → evaluation with distance-based metrics.

### Pipeline Architecture

```
Pexels videos (5 clips)
    ↓
Frame extraction (every 15th frame, ~2 FPS)
    ↓
Auto-labeling with YOLOv8x (pretrained on COCO)
    ↓  Filter: car, truck, bus, motorcycle → single class "vehicle"
    ↓  Clean: remove small/large/low-conf boxes
    ↓  Light manual cleanup
    ↓
YOLO format dataset (train 90% / val 10%)
    ↓
Fine-tune YOLOv8n (50 epochs, 640px, AdamW)
    ↓
Evaluate on held-out eval clip
    ↓  Distance estimation from bbox pixel size
    ↓  Metrics per band: 0-200m, 200-400m
    ↓
Output: metrics table + video + examples
```

## Key Decisions

### 1. Auto-Labeling Strategy (the heart)

**Why YOLOv8x for labeling, YOLOv8n for training?**

I used the largest YOLO model (YOLOv8x, 68.2 mAP on COCO) to generate pseudo-labels, then trained the smallest model (YOLOv8n, 37.3 mAP on COCO) on those labels. This is a form of **knowledge distillation** — a larger teacher model generates labels for a smaller student model. The student model can then learn to detect vehicles in aerial views, which is a domain shift from COCO's ground-level perspective.

**Class merging:**

COCO has separate classes for car (2), motorcycle (3), bus (5), and truck (7). I merged all into a single `vehicle` class (0) since the task specifies single-class detection.

**Filtering pipeline:**

1. **Confidence threshold = 0.25** — deliberately low to catch more vehicles, accepting some noise. Better to have a noisy-but-comprehensive dataset than a clean-but-sparse one.
2. **Min bbox area = 0.05%** — removes tiny noise detections (dust, artifacts).
3. **Max bbox area = 15%** — removes oversized false positives (entire road segments misclassified).
4. **NMS IoU = 0.5** — standard duplicate removal.

### 2. Manual Cleanup

After auto-labeling, I visually inspected a sample of frames from each clip:

- **Removed**: Shadow detections, road markings misidentified as vehicles, duplicate boxes on single vehicles.
- **Added**: Clearly visible vehicles that the auto-labeler missed (usually partially occluded or at unusual angles).
- **Adjusted**: Boxes that were too tight or too loose on vehicle boundaries.

Cleanup affected approximately 5-10% of labels. The changes are documented in `data/labels_clean/cleanup_log.txt`.

### 3. Training Configuration

- **Model:** YOLOv8n (nano) — chosen for edge deployment relevance. If this model needs to run on a drone's compute module, nano is the realistic choice.
- **Image size:** 640px — standard YOLO training size, good balance of detail vs compute.
- **Epochs:** 50 with patience=15 — early stopping prevents overfitting on a small dataset.
- **Augmentation:** Default YOLO augmentation (mosaic, mixup, copy-paste, flip, scale) — critical for a small dataset to improve generalization.
- **Optimizer:** AdamW with lr=0.001 — stable convergence.
- **Batch:** 16 (reduce to 8 if GPU memory is limited).

### 4. Distance Estimation

**Assumptions (stated explicitly):**

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| Reference object size | 4.5 m (car length) | Average sedan length |
| Camera horizontal FOV | 70° | Typical drone/surveillance camera |
| Camera model | Pinhole | Simplified, no lens distortion |

**Formula:**

```
focal_length_px = image_width / (2 × tan(FOV/2))
distance_m = (object_size_m × focal_length_px) / bbox_width_px
```

**Limitations:**
- Assumes all detected objects are cars (4.5m). Trucks are longer, motorcycles shorter — introduces error.
- Pinhole model ignores lens distortion.
- Top-down vs oblique angle changes the apparent size differently.
- These are rough estimates, not precise measurements.

### 5. Evaluation Protocol

- **Eval clip is completely held out** — not used for training, threshold tuning, or model selection.
- Ground truth for eval clip is generated by YOLOv8x (same auto-labeling pipeline as train, but kept separate).
- Metrics are computed per distance band: detections matched to GT by IoU ≥ 0.5, then separated by estimated distance.

## What I'd Improve With More Time

1. **Better auto-labeling:** Use multiple models (YOLO + DETR + Faster R-CNN) and take consensus — reduces individual model biases.
2. **Active learning loop:** Train initial model → run on unlabeled data → manually verify high-uncertainty predictions → retrain.
3. **Tiling for small objects:** Split high-res frames into overlapping tiles, detect on each tile, merge results. Critical for 200-400m range.
4. **Tracking-based augmentation:** Use object tracking (DeepSORT) between frames to interpolate missed detections and smooth labels.
5. **Synthetic data:** Generate aerial views of vehicles using cut-paste augmentation on real backgrounds at various scales and angles.
6. **Better distance estimation:** Use camera metadata (altitude, pitch angle) if available, or train a depth estimation model.
7. **Model ensemble:** Use TTA (Test-Time Augmentation) or model ensemble for evaluation.
